
# Чеклісти для тестування

## Модуль: Адмін-панель (Додавання та редагування пари в розклад)

### Функціональність
- [ ] Перевірка успішного відкриття вікна "Додати пару" при кліку на вільний слот у сітці розкладу.
- [ ] Перевірка успішного відкриття вікна "Редагування пари" при кліку на вже зайнятий слот або іконку олівця.
- [ ] Перевірка вибору викладача з випадаючого списку (Dropdown).
- [ ] Перевірка вибору предмета з випадаючого списку.
- [ ] Перевірка вибору типу заняття (Лекція / Практика тощо) та кількості годин.
- [ ] Перевірка роботи чекбоксу "Групова" для створення спільної лекції для кількох груп.
- [ ] Перевірка успішного збереження форми та додавання пари в розклад при заповненні всіх обов'язкових полів.
- [ ] Перевірка закриття вікна без збереження внесених змін при натисканні на кнопку "Скасувати" або хрестик.

### Негативні сценарії
- [ ] Перевірка спроби збереження форми з незаповненими обов'язковими полями (наприклад, без викладача або предмета).
- [ ] Перевірка створення пари на час, коли обрана аудиторія вже зайнята іншою групою (очікується помилка "Аудиторія зайнята").
- [ ] Перевірка створення пари для викладача, який у цей самий час уже проводить заняття в іншій групі.
- [ ] Перевірка успішного збереження після виправлення конфлікту (зміна аудиторії/викладача на вільні після виникнення помилки).
- [ ] Перевірка можливості зміни належності пари до групи або додавання додаткової групи під час редагування існуючої пари (тест-кейс на виявлений баг).

### UI/UX
- [ ] Перевірка наявності та коректності плейсхолдерів і підписів до полів форми (Викладач, Предмет, Тип, Год тощо).
- [ ] Перевірка візуального відображення помилок валідації (наприклад, підсвічування червоним кольором незаповнених обов'язкових полів).
- [ ] Перевірка клікабельності та підсвічування кнопок "ЗБЕРЕГТИ" та "СКАСУВАТИ" при наведенні курсора (hover-ефект).
- [ ] Перевірка коректного відображення модального вікна (відсутність накладання елементів чи виходу тексту за межі форми) при роздільній здатності 1920х1080.
